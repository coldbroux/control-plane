Create the `semantic_modeling` schema and its two tables: `candidate_terms` (durable modeling object) and `candidate_term_observations` (per-source-term observation junction). Apply via a numbered migration in `peddlerr/api/db/migrations/`.

This task is the schema foundation for every other task in the PBI. Nothing else lands until the tables exist.

### What if we didn't do this step?

**Downstream Consequences:** Tasks 2–8 cannot land — the candidate-term memory layer cannot exist without its tables.

**Technical Debt:** The architectural contradiction (cross-source skill with no cross-source memory) persists.

**Alternative Approaches:** None viable. Per-system playbook files were considered and rejected — they scatter signal with no queryable rollup.

**Risk Assessment:** Low — additive schema work, no impact on existing tables.

## Description

Author migration `peddlerr/api/db/migrations/042_add_semantic_modeling_candidate_terms.sql` (042 is the next available number; 041 is the latest existing migration). The migration creates:

- Schema `semantic_modeling`.
- Tables `semantic_modeling.candidate_terms` and `semantic_modeling.candidate_term_observations` per the Requirements section.
- A CHECK constraint enforcing resolution-column exclusivity by status.
- A partial unique index on `normalized_term` for active states.
- Supporting btree indexes on FK columns and `status`.

The migration must be safe under standard Postgres migration semantics (idempotent within a single deployment when re-applied via the lifecycle tooling).

## Requirements

1. `semantic_modeling` schema exists.
2. `semantic_modeling.candidate_terms` columns:
   - `candidate_term_id` uuid PK default `gen_random_uuid()`
   - `normalized_term` text NOT NULL
   - `candidate_role` text (nullable; e.g. `HealthcareOffering`, `LocationType`)
   - `status` text NOT NULL DEFAULT `'candidate'` CHECK (status IN (`'candidate'`, `'deferred'`, `'promoted'`, `'merged'`, `'rejected'`, `'obsolete'`))
   - `evaluation_notes` text (nullable; populated at any state transition)
   - `first_seen_version_id` uuid NOT NULL REFERENCES `core_semantics.semantic_model_versions(id)`
   - `last_seen_version_id` uuid NOT NULL REFERENCES `core_semantics.semantic_model_versions(id)`
   - `merged_into_candidate_id` uuid REFERENCES `semantic_modeling.candidate_terms(candidate_term_id)`
   - `resolved_concept_key` text (nullable; semantic reference to `core_semantics.canonical_concepts.concept_key`; FK only if existing schema permits)
   - `resolved_at` timestamptz (nullable)
   - `created_at` timestamptz NOT NULL DEFAULT `now()`
   - `updated_at` timestamptz NOT NULL DEFAULT `now()`
3. CHECK constraint on `candidate_terms`:
   - `status = 'merged'` → `merged_into_candidate_id IS NOT NULL AND resolved_concept_key IS NULL AND resolved_at IS NOT NULL`
   - `status = 'promoted'` → `resolved_concept_key IS NOT NULL AND merged_into_candidate_id IS NULL AND resolved_at IS NOT NULL`
   - other statuses → `merged_into_candidate_id IS NULL AND resolved_concept_key IS NULL AND resolved_at IS NULL`
4. Partial unique index: `UNIQUE (normalized_term) WHERE status IN ('candidate', 'deferred', 'promoted')`.
5. `semantic_modeling.candidate_term_observations` columns:
   - `observation_id` uuid PK default `gen_random_uuid()`
   - `candidate_term_id` uuid NOT NULL REFERENCES `semantic_modeling.candidate_terms(candidate_term_id)`
   - `vocabulary_term_id` uuid NOT NULL REFERENCES `core_semantics.vocabulary_terms(id)`
   - `version_id` uuid NOT NULL REFERENCES `core_semantics.semantic_model_versions(id)`
   - `observed_at` timestamptz NOT NULL DEFAULT `now()`
6. `UNIQUE (vocabulary_term_id, version_id)` on `candidate_term_observations`.
7. Btree indexes: `candidate_terms(status)`, `candidate_terms(merged_into_candidate_id)`, `candidate_term_observations(candidate_term_id)`, `candidate_term_observations(version_id)`.
