## Description

Update the skill so that Check 2 explicitly:

1. Runs Q8 first.
2. Walks each bucket in turn:
   - `never_seen` → reason from scratch (existing flow).
   - `candidate` → transient state from a current-run recording; treat as `never_seen` for evaluation.
   - `deferred` → display prior `evaluation_notes` and `cross_source_count`. Ask: does the prior reason still hold, or has accumulating evidence changed it?
   - `merged` → display target candidate. Do not propose a new evaluation; the merge target should appear in its own bucket.
   - `rejected` → do not re-propose. Surface the rejection rationale in case it's worth revisiting, but no automated re-evaluation.

4. Section 2 (promotions) writes `promote` decisions, with the heuristic warning honored.
5. Merges are an explicit Section in the skill output (not implicit). When the agent recognizes that two candidates should collapse into one, it produces a `merge` decision targeting the surviving candidate.
