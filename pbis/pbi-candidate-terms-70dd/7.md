# 7 Backfill 38 Adena deferrals

Back to task list: `tasks.md`

## Overview

Backfill the 38 deferred terms produced during the `pbi-research-orch-2e0b` Task 8 E2E run as `status='deferred'` candidates in the new schema, with one Adena observation each. Provides immediate state for testing Q8 and confirms the lifecycle works against real evaluations rather than synthetic fixtures.

### What if we didn't do this step?

**Immediate Impact:** The new schema starts empty. Q8 produces an empty `deferred` bucket on the next Phase 2 run.

**Downstream Consequences:** Task 8 E2E validation would have to re-run Phase 2 from scratch and re-derive deferrals before the accumulating-memory behavior could be exercised.

**Technical Debt:** Discards the prior session's reasoning. The 38 evaluations took non-trivial agent time.

**Alternative Approaches:** Re-run Phase 2 against Adena and let the new pipeline produce the records naturally. Equivalent end state but loses the prior rationale captured in the original transcript.

**Risk Assessment:** Low — explicit one-shot backfill.

## Description

Author a one-shot script `peddlerr/workbench/scripts/backfill_adena_deferrals.py` that:

1. Reads a hand-curated input file mapping each of the 38 terms to its `evaluation_notes` (the prior rationale).
2. For each term:
   - Look up the corresponding `vocabulary_terms` row in the current Adena `is_current` model version.
   - Insert a `candidate_terms` row with `status='deferred'`, `normalized_term` set to the term's normalized form, `first_seen_version_id = last_seen_version_id = current Adena version`, `evaluation_notes` from the input file.
   - Insert a `candidate_term_observations` row linking to the vocabulary term.
3. None of the 38 are promoted automatically — single-source observations don't satisfy the heuristic.
4. The input file is committed alongside the script (e.g. `peddlerr/workbench/scripts/data/adena_deferrals_backfill.tsv` or similar) so the backfill is reproducible.
